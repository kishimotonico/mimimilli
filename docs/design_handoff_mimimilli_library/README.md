# Handoff: mimimilli — Library モード UI

## Overview
**mimimilli** は ASMR / 環境音 / ボイス作品を管理・再生するデスクトップ風 **Web アプリ**。
本パッケージは、既存の **File Explorer モード**（物理ファイルを Finder 風カラムで整理）に並ぶ
もう一つの第一級モード **Library モード** の UI デザインをまとめたもの。

Library モードは「登録済み作品」に対する **仮想ファイルシステム**。物理パスに縛られず、
サークル別 / CV 別 / タグ別 / スマートフォルダーなど任意の「軸（ピボット）」で作品を並べ替えて
ブラウズできる。再生エンジンは File / Library 両モードで**共通・常駐**しており、Library で音声を
重ねて再生しながら File モードで物理ファイルを整理する、といった併用が成立する。

> 実装優先度: **Library 機能（物理ファイルでない、登録済みメタデータ上の操作）の方が高い**。
> File Explorer モードは既存実装/別デザインがある前提。本パッケージは Library を中心に、
> 共通シェル（メニューバー・左ナビ・再生バー）と File モードとの接続点を記述する。

---

## About the Design Files
このバンドル内の HTML / JSX / CSS は **HTML で作られたデザインリファレンス**であり、
意図する見た目・挙動を示すプロトタイプです。**そのまま本番コードにコピーするものではありません。**

タスクは、これらのデザインを **ターゲットのコードベースの既存環境**（Electron + React / Tauri +
任意フレームワーク / SwiftUI など）で、その確立されたパターン・ライブラリを使って**再現すること**です。
まだ環境が無い場合は、本アプリに最適なフレームワークを選定して実装してください
（デスクトップアプリ化の構想もあるが、当面は Web アプリとして動作させる想定）。

プロトタイプは React + Babel(standalone) をブラウザ上で直接トランスパイルして動かしています。
これは「あくまでデザインを動かすため」の構成で、本番のビルド構成とは無関係です。

## Fidelity
**High-fidelity (hifi)。** 色・タイポgrafi・余白・状態（hover/selected/focused）まで作り込んだ
ピクセル精度のモック。`milli/tokens.css` のデザイントークンをそのまま移植し、各コンポーネントを
忠実に再現してください。レイアウト寸法・色 hex（oklch）・フォント指定は本 README と各 CSS に明記。

## ⚠️ Web アプリ化のための必須修正（プロトタイプとの差分）
プロトタイプはデスクトップ枠（ウィンドウ chrome）を持たず、すでに **アプリ独自のメニューバー**に
置き換え済み。実装時も以下を踏襲すること:

1. **OS ウィンドウのメニューバー / 信号機ボタンは出さない。** 代わりにアプリ内に
   トップバー（`.mll-bar`）を設ける。
2. **モード切替（ファイル / ライブラリ）は左ナビ**に置く（トップバーではない）。常に現在モードが
   選択状態で表示される。
3. **再生は両モード共通・常駐。** モードを切り替えても再生は途切れない。再生状態はグローバルに保持。

ルートは固定の 4 行グリッド（`.mle-frame.is-lib`）。横は 2 カラム（左ナビ 64px + 本体 1fr）。

```

│  ① トップバー  .mll-bar                       (高さ 48px)   │  ← 全幅
├──────┬────────────────────────────────────────────────────┤
│      │  ② アドレスバー .mle-addr            (高さ 38px)     │
│ ⓪左  ├────────────────────────────────────────────────────┤
│ ナビ │                                                     │
│.mle- │  ③ 本体 .mle-body  (カラムビュー + プレビュー) 1fr   │
│ side │                                                     │
│ 64px ├────────────────────────────────────────────────────┤
│      │  ④ 再生バー  (状態で 56 / 78 / 96px)                │
└──────┴────────────────────────────────────────────────────┘
```

CSS グリッド定義（`columns.css` / `explorer.css`）:
- `.mle-frame.is-lib`              → `grid-template-rows: 48px 38px 1fr 96px;`
- `.mle-frame.is-lib.is-mixer-single` → `48px 38px 1fr 78px;`（再生 1ファイル時）
- `.mle-frame.is-lib.is-mixer-empty`  → `48px 38px 1fr 56px;`（停止時）
- 左ナビ `.mle-side` は `grid-column: 1; grid-row: 3 / span 2;`（②③④の左に縦に通る）
- トップバー `.mll-bar` は `grid-column: 1 / 3;`（全幅）

## ⓪ 左ナビ `.mle-side`（64px・アイコン+ラベルの縦レール）

2. **横断サーフェス**（区切り線で区切る）

   - `ピン留め`（icon: bookmark）
3. スペーサー（`flex:1`）

## ② アドレスバー `.mle-addr.is-lib`（高さ 38px）
- 戻る/進む/再読込（`.mle-navbtn`）
- **パンくず** `.mle-crumbs`: 階層を `/` 区切りで表示。**階層移動はここが唯一の経路**（カラムでは遡らない）。
  例: `ライブラリ / サークル / 夜想曲`、ファイル: `/Volumes / Audio / asmr / 2024 / 12月 / RJ01234567`
- **ビュー切替** `.mle-addr__vtoggle`: カラム(gridS) / リスト(list) / グリッド(grid)。既定はカラム。
- 並び替え(sort) / その他(more)
- ※ Library モードではトップバーに検索があるため、アドレスバー内の検索は出さない。

## ③ 本体 `.mle-body` → カラムビュー `.mle-cols` + プレビュー `.mle-prv`
`.mle-cols` は横並び flex（`overflow-x:auto`）。右端に伸縮するプレビュー（`flex:1; min-width:360px`）。

### Library は常に 3 ペイン（軸レール → コンテンツ列 → プレビュー）
深い分類でも**カラムを増やさない**のが核。

#### col0 = 軸レール `.mle-col.is-axis`（幅 192px）
作品の並べ替え軸（ピボット）を選ぶ。3 グループに区切り（`.mll-axisgroup`、見出し `.mll-axisgroup__hd`）:
- **ビュー**: すべての作品(347) / 最近再生(18) / 最近追加(23, バッジ) / お気に入り(42) / 未再生(12) / ファイル欠損(2)
- **分類軸**: サークル(32) / CV(28) / シリーズ(24) / カテゴリ(8) / タグ(62) / 追加日
- **スマートフォルダー**: 就寝用ロング(9) / 雨音—今夜(5) / 月白あかね 全件(6) / … / `+ 新規スマートフォルダー`

各行 `.mll-axis`: アイコン + 名前 + (バッジ) + 件数 + chevron。選択 `.is-on`、フォーカス時 `.is-on.is-focused`（黒地白文字）。

#### col1 = コンテンツ列 `.mle-col.is-content`（幅 300px）
選んだ軸の結果。2 つの見え方:
- **単純ビュー**（すべて/最近再生/お気に入り等）→ そのまま作品リスト
- **分類軸/スマート**（サークル等）で1段降りた時 → **ドリルヘッダー `.mle-drill`** に親を畳んで
  その下に子（作品）を重ねる。＝「col2 を別カラムにせず col1 の上に積む」。新カラムは開かない。
  - ドリルヘッダー: 戻る矢印 + 軸名（小） + `/` + 値（夜想曲 など） + 件数 + chevron。下に
    サブ見出し `.mle-drill__sub`（例「この サークル の作品 / 3」）。

**作品行 `.mll-wrow`**（Library のシグネチャ。File モードの薄い行と違いサムネ付き）:
- 32px カバーサムネ + 本体(タイトル `--font-jp` 12px / サブ `--font-mono` 9.5px: サークル·tr数·尺) + 右メタ
- 右メタ: 再生中なら `▸ 再生中`（アクセント地）/ お気に入りなら星 / それ以外は追加日
- 選択 `.is-on`、フォーカス `.is-on.is-focused`（黒地白文字）

#### プレビュー（右・伸縮）
軸/選択により出し分け:
- **作品選択時 → リッチ詳細**（後述）
- **タグ多軸など一覧時 → AxisLanding**: スライス概要 + 一括操作 + 上位作品グリッド
- **スマートフォルダー選択時 → ルールビルダー**（後述）

### タグ軸だけ特別: 多軸（積集合）
col1 はタグ一覧（`.mll-tagrow`、チェックボックス + 色スウォッチ + 件数）。上部に選択中タグを
**AND チップ**で並べる `.mll-tagband`（`雨音` AND `長時間` … + ヒット件数）。結果は別カラムにせず
**プレビュー側 (AxisLanding)** に出す。物理FSでは不可能な立体的絞り込み。

## リッチ作品詳細（Library プレビューの主役）`LibWorkDetail`
File モードのプレビューが「フォルダー内訳 + 販売リンク」なのに対し、Library は**再生体験**に焦点:
- **ヒーロー**: 大きいカバー + キッカー(登録済 / 追加日 / 最終再生) + タイトル + サブ + (tracks·尺·★レーティング) + タグ + アクション（最初から再生 / Ch に追加 / キュー / ♥ / …）
- **統計 4 枚** `.mll-stats`: 再生回数 / 累計時間 / 最終再生 / 完聴率（`--font-mono` 14px）
- **メモ** `.mll-notes`: 付箋風（淡い黄）。ユーザー自由記述。
- **トラック表 `.mll-rtrk`**: # / タイトル / 長さ / **再生回数** / 最終再生 / 行アクション。再生中行 `.is-now`。
- **関連 `.mll-related`**: 同サークル/同タグの作品カード（カバー + 再生回数バッジ + タイトル + メタ）。

## スマートフォルダー = 保存済みクエリ `SmartFolderPreview`
- col0 でスマートフォルダーを選択 → col1 = **そのフォルダーにマッチした作品一覧** → プレビュー = ルール。
  （col0 と col1 で同じ一覧を出さないこと。col0=選択肢、col1=中身。）
- ルールビルダー `.mll-smart`: `SMART` バッジ + 名前 + 再評価/保存。
  ルール `.mll-smart__rule` を `WHERE / AND / AND NOT` の接続詞付きで縦に列挙。
  例: `長さ ≥ 3:00:00` / `カテゴリ = 環境音 OR ASMR` / `タグ ∋ 睡眠用 OR 長時間 OR 焚き火` / `AND NOT タグ ∋ R-15`
  `+ 条件を追加`。フッター: `N 件マッチ · 累計尺` + 並び順 + 名前変更/削除。

## ④ 再生バー（両モード共通・常駐）— 状態で 3 段階
「**基本は 1 ファイル再生**、重ねたい時だけ最大 3ch」という前提。

| ch数 | 表示 | 高さ | クラス |
|---|---|---|---|
| 0 | 細いヒントバー（「ファイル/作品をクリックして再生」+ Shift で重ねる） | 56px | `.mle-bar1.is-empty` |
| 1 | **コンパクト単独トランスポート（既定）** | 78px | `.mle-bar1` |
| 2–3 | マルチ ch ミキサー | 96px | `.mle-mixer` |

### 単独トランスポート `.mle-bar1`（1ファイル時）
左: 46px カバー + トラック名/作品名 + ♥。 中央: ⏮ / ⏯(38px丸) / ⏭。
スクラブ: 経過時刻 + バー（アクセント塗り + つまみ） + 総尺。
右: ループ / 音量スライダー / 区切り / **「重ねて再生」**ボタン(`.mle-bar1__stack`、これを押すとマルチ ch へ昇格) / キュー / 拡大。

### マルチ ch ミキサー `.mle-mixer`（2–3 ファイル）
左にマスター（`同時再生 / N ch / 全停止`）。各 ch `.mle-ch` は色ストライプ(coral/sky/leaf/mustard/plum) +
CH番号 + サムネ + トラック/作品 + S(ソロ)/M(ミュート)/×、スクラブ、再生/音量/ループ。`< 3ch` の時のみ「チャンネル追加」スロット。

## File モード（接続点）— 深い階層は「受動スタック」
File モードも同じカラムビューだが、**階層が深くなってもカラムを無限に増やさない**:
- 末尾 2 階層 + プレビューだけ全幅を保つ。
- それ以前の祖先カラムは左端で **受動的に重なってスタック** `.mle-colstack`（幅 38px、
  カードの小口が数枚ずれて重なった見た目のみ。ラベル/件数バッジは出さない）。
- **クリックでの再展開はしない。階層移動はパンくず。**
- 実 UI のアニメーション: 子を開くと新カラムが右からスライドイン、はみ出た最古カラムが左の束へ吸い込まれる。

## Interactions & Behavior
- **モード切替**（左ナビ ファイル/ライブラリ）: 本体カラム構成を切替。再生は維持。
- **軸選択**（col0）: col1 の内容を差し替え。単純ビューは作品リスト、分類軸は1段降りでドリルヘッダーに畳む。
- **タグ多軸**: タグのチェックで AND 積集合、ヒット件数と結果を即時更新。
- **作品クリック=単独再生**（既定 1ch）／**Shift+クリック or「重ねて再生」=ch 追加**（最大 3ch）。
- **スマートフォルダー**: ルール編集で結果即時再評価。`+ 新規` で作成フロー（モーダル想定・本パッケージ未設計）。
- **パンくず**: 唯一の階層遡上手段。
- **カラムスタック**: 受動表示。クリック非対応。
- ホバー/選択/フォーカスは各 CSS の `:hover` / `.is-on` / `.is-on.is-focused` を踏襲。
- ダーク/アクセント色は `tokens.css` の `.ml-dark` と `--acc` 系を差し替えるだけで全体追従。

## State Management（最小モデル案）
- `mode: 'files' | 'library'`（グローバル）
- `playback: { channels: Channel[] (0..3), masterPlaying }`（**グローバル・モード非依存**）
  - `Channel { id, workId, trackId, positionSec, durationSec, volume, loop, solo, mute, color }`
- Library: `activeAxis`（軸 id）, `drill`（降りた facet 値）, `selectedTags[]`, `selectedWorkId`, `sort`
- File: `path[]`（カラムスタックは派生 = 末尾2件以外を畳む）
- 作品ごとのユーザーデータ: `playCount, lastPlayedAt, totalListenedSec, rating(0-5), favorite, pinned, note`（永続化対象）
