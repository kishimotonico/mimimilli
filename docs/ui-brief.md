# mimikago UI ブリーフ

このドキュメントは、mimikago の UI を新しく設計してもらうための依頼書です。画面構成・配色・配置・コンポーネント分割などの UI 上の判断は、すべてあなた（デザイナー）に委ねます。本書は「このアプリが何を扱い、ユーザーが何をしたいか」だけを伝えます。

## 1. このアプリは何か

ローカルに保存した音声作品（DLsite や FANZA からダウンロードした ASMR 等）を、自分一人で管理・再生するためのローカル Web アプリです。

ただし「再生するだけのアプリ」ではありません。作品フォルダーには音声以外のファイル（カバー以外のイラスト、特典 PDF、テキストファイル、紹介動画など）が一緒に同梱されることが多く、**作品フォルダー全体を快適に閲覧できるビューワー** であることも重要です。トラックリストだけでなく、物理的なフォルダー構造をたどって個々のファイルを眺められる必要があります。

- 利用者は本人ひとり。マルチユーザーや共有・公開の概念はありません
- 作品数は数百〜数千規模。1 作品あたりトラック数本〜数十本
- 主な利用環境は PC のブラウザ。将来的に同一 LAN 内のタブレット等からも使う想定

## 2. 扱うデータ（情報構造）

### 作品 (Work)

- タイトル、カバー画像、追加日、物理パス
- **複数の URL**（ラベル付き。販売ページ・公式サイト・SNS など、1 件〜5 件程度）
- **複数のタグ**（後述）
- **1 〜 複数のプレイリスト**（SE 有無やフォーマット違いのバリアント。通常は 1 つだけ）
- **作品フォルダーに同梱される物理ファイル群**（後述）
- エラー状態（メタファイル不正、参照先音声ファイルの欠損、非対応フォーマット など）
- 再生位置・ブックマーク・最終再生日時（将来扱う情報）

### トラック (Track)

- タイトル、再生時間
- 参照ファイルパス
- 単一ファイル形式の場合は再生区間（start/end の秒数）

### 物理ファイル (File)

作品フォルダーには、トラックとして登録されている音声ファイル以外にも様々なファイルが入っています。サブフォルダーで階層構造になっていることも普通です。

- 音声ファイル（トラックに登録されているもの／されていないもの両方ある）
- 画像ファイル（カバー以外のイラスト、特典画像。複数枚あることが多い）
- 動画ファイル（紹介動画、特典動画など）
- テキストファイル、PDF（説明書、特典テキスト、台本など）
- それ以外のファイル（情報だけ見えればよい）

ユーザーはこれらのファイルをアプリ内から直接プレビュー／再生したいと考えています。

### タグ (Tag)

タグは 2 種類が混在します。

- **フラットタグ:** 単純な文字列。例: `バイノーラル`、`耳かき`、`R-15`
- **Annotated タグ:** `カテゴリ/値` 形式。例: `cv/ひなたゆか`、`サークル/ほげふが`、`シリーズ/○○物語`

1 つの作品に対して、両者あわせて数十個のタグが付くことが普通です。Annotated タグは「どのカテゴリに属する値か」が一目で分かることが重要です。

## 3. ユーザーは何をしたいか（ユースケース）

優先度の高い順に並べます。

### 3.1 コレクションを眺めて、今夜聴くものを選びたい

- カバー画像を中心に、大量の作品を視覚的にブラウズする
- タイトル・タグでの絞り込み
- 並び替え（追加日、タイトル、合計再生時間）

### 3.2 作品の中身を確認して、特定のトラックから再生したい

- 作品のメタ情報（タグ、URL、トラック構成、合計再生時間）をすぐ確認できる
- 任意のトラックを選んで、そこから再生を始める

### 3.3 再生中、長時間（数十分〜数時間）快適に聴き続けたい

- 一時停止 / シーク / 前後トラック移動 / 音量
- ±10 秒のジャンプ（聴き逃した箇所の戻り、ASMR で重要）
- 1 トラックのループ再生
- 再生中に他の作品を眺めたり検索したりすることもある（再生は途切れない）

### 3.4 作品フォルダーの中身を物理ファイル単位で覗きたい

トラックリストとは別に、作品フォルダーの中身そのものをブラウズできることが重要です。

- フォルダーの階層構造をたどる
- 画像をプレビューする（複数枚あれば次々と見たい）
- 動画を再生する
- テキストや PDF の中身を確認する
- 音声ファイルもこの場から再生できる（トラック未登録のおまけ音声なども）

### 3.5 作品を整理したい

- タグを追加・削除する
- 作品に紐づく URL を外部ブラウザで開く（販売ページに飛ぶ等）

### 3.6 新しく追加したフォルダーを取り込みたい

- ルートフォルダーを再スキャンする
- 新規に検出された作品候補を 1 件ずつ確認しながら登録する（タイトルを直す・スキップするなど）

### 3.7 設定を変えたい・初回セットアップ

- ルートフォルダーを指定／変更する
- 手動でフルスキャンを実行する（前回スキャン日時も把握したい）
- 初回起動時はルートフォルダー指定〜初回スキャンの流れに案内する。ここは凝らずに、迷わず通り抜けられれば十分

## 4. 制約・前提

- **デバイス:** デスクトップブラウザ優先。タブレット幅でも破綻しない程度であれば嬉しい
- **想定シーン:** 夜間にヘッドホンで長時間使うことが多い
- **再生の常駐:** 一度再生を始めたら、別の作品を見たり画面を切り替えたりしても再生は途切れない。今何を再生しているか、ユーザーが常に把握できる必要がある
- **キーボード操作:** スペースキーで再生/一時停止のトグルができると嬉しい
- **エラー作品の存在:** メタファイルの不正やファイル欠損を持つ作品が混ざる。一覧上で「これはエラーがある」と気付ける必要がある
- **空状態:** 作品 0 件、検索ヒット 0 件 などの状態が存在し、それぞれ何をすればいいかユーザーに伝えたい
- **UI 文言:** 日本語

## 5. 成果物

- **形式:** HTML + CSS の静的モック（複数画面）
- **画面の選定:** どの画面を作るかはあなたの判断に任せます。第 3 章のユースケースが一通り体験できることが目安
- **技術:** フレームワーク不要。素の HTML/CSS、または Tailwind など任意の CSS 手法を使ってかまいません
- **デザインの自由度:** 配色、タイポグラフィ、画面分割、ナビゲーション構造、レイアウト、インタラクション表現はすべてあなたに委ねます
