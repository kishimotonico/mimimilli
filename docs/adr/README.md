# ADR (Architecture Decision Records)

整理された設計判断の記録を置く場所。

日々の作業ログや実装タスクの記録は [`../issues/`](../issues/) を使う。

## いつ書くか

後から来た人・エージェントが親切心で「直したくなる」意図的な選択を ADR にする。将来の実装判断を縛る決定（技術スタック・永続化方式・境界の引き方・データ所有権・性能や配布に関わる方針）が対象。命名変更や作業手順の決定は issues で足りる。

## 命名規則

`NNNN-short-kebab-title.md`（連番4桁、英語の短いタイトル）。

## 不変の記録として扱う

ADR は原則として本文を書き換えない。状況が変わった場合は、ファイル末尾に「## 現状（YYYY-MM-DD）」節を追記するか、新しい ADR を書いて既存の決定を上書きする（本文中でその ADR にリンクする）。

## テンプレート

```markdown
# ADR-NNNN: タイトル

- ステータス: 承認 / 検討中 / 却下 / 廃止
- 日付: YYYY-MM-DD
- 関連: 関連する ADR や文書へのリンク

## 文脈

なぜこの決定が必要になったか。

## 決定

何を決めたか。

## 帰結

この決定によって何が可能になり、何を諦めたか。
```
