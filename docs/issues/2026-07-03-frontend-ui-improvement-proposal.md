# フロントエンドUI改善提案

Status: done — 残タスクは BACKLOG.md へ

作成: 2026-07-03 / Claude（agent-browser実地調査 + コード調査を統合）

## 前提

- agent-browser（`--session mimimilli-design`）で全画面を実際に操作して確認した。コード側は client/ 全体を調査済み
- モックを正典とするフェーズは終了しており、本提案は動いているフロントエンドを出発点にする。モックとの差分自体は課題として扱わない
- 再生UIの「バー + ポップアップ + 全画面」の2層+α構成（`a2b4e4a`）は意図的な設計として維持する
- 未配線のUI（左ナビの履歴/ピン留め、ビュー切替、スマートフォルダーの条件追加など）は将来機能の枠なので本提案のスコープ外。ただし「押せるのに無反応」への手当てだけ T5 に含めた

## 現状診断の要約

レイアウト骨格（4行グリッド、3ペイン、軸レール、ドリルヘッダー、タグANDチップ）とデザイントークンは完成度が高い。弱いのは操作系で、根っこは次の3つに集約される。

1. ボタンの共通コンポーネントが存在しない。`mll-fab` / `mle-icbtn` / `mle-navbtn` / `mll-bar__icbtn` / `mle-side__btn` / `mle-prv__edit-action` の6種のCSSクラスを都度直付けしており、箱のサイズはCSS側・アイコンの描画サイズは呼び出し側の `size` prop（11〜16px）と二重管理。余白や文字位置のばらつきはここから生まれている
2. 編集UIが「インライン展開 + テキストボタン」方式。タイトル編集とタグ編集が同時に開けて「保存/キャンセル」が2組並ぶ。テキストボタンは10pxの `mle-prv__edit-action` に集中
3. タグ候補がネイティブ `<datalist>`。件数制限も絞り込み制御もスタイル制御もできず、入力ゼロで全タグ（現在19件）が出る

また実バグを1件発見した: 再生バー表示中、バーの本体（`.mle-bar1__body`）が下のコンテンツへのクリックを奪い、軸レール最下部の「タグ」行がクリックできなかった（T4に修正を含む）。

## T1. ボタン体系の整備（優先度: 高・他テーマの土台）

`shared/ui/` に共通コンポーネントを新設し、既存の6種CSSクラスを吸収する。

- `IconButton`: `size="sm|md|lg"`（箱 26/30/38px、アイコン 14/16/20px を対で固定し二重管理を解消）、`aria-label` 必須、`title` によるツールチップ標準装備
- `Button`: `variant="primary|ghost|quiet"`。primary は現行 `mll-fab`（pill・アクセント地）を引き継ぐ

アイコン化の方針は「役割で線を引く」:

- プライマリアクション（再生系）は pill + 短ラベルを維持する。全部アイコンにすると初見の可読性が落ちる
- セカンダリ以下（編集・補助・ナビ）はアイコンボタン + ツールチップに寄せる

具体的な置き換えマップ:

| 現状                           | 置き換え                                        | 場所                      |
| ------------------------------ | ----------------------------------------------- | ------------------------- |
| 「編集」テキスト               | 鉛筆アイコン                                    | `PreviewPane.tsx:199`     |
| 「タグを編集」テキスト         | T2で編集導線ごと再設計                          | `PreviewPane.tsx:267`     |
| 「保存」「キャンセル」×2組     | T3でポップオーバー化に伴い消滅                  | `PreviewPane.tsx:189-257` |
| 「最初から再生」テキスト       | ↺ アイコン（続きから再生の隣）                  | `PreviewPane.tsx:282,292` |
| 「再生中の作品を表示」テキスト | 照準アイコン + ツールチップ                     | `PopupContent.tsx:235`    |
| 「縮小」テキスト               | minimize アイコン                               | `FullScreenPlayer.tsx:72` |
| 「▸ 再生中」チップ（作品行）   | イコライザーアニメ（3本バー）+ アクセント色維持 | `WorkRow.tsx:58`          |

アイコンは既存の自前セット（`shared/ui/Icon.tsx`、24pxグリッド線画）に追加する形でよい。lucide等の導入は不要。

## T2. タグ編集UIの刷新（優先度: 最高）

`<datalist>` を廃止し、カスタムコンボボックス `TagCombobox` を新設する。

- 入力が空のときは候補を出さない（または「最近使った5件」のみ）
- 1文字以上で部分一致フィルタ、表示は上位8件まで。各候補に使用件数を添える
- 完全一致が無い入力には「"○○" を新規作成」行を出す
- Enter/クリックで即チップ追加、Backspaceで末尾チップ削除。フォーカスリング・ハイライトはトークン準拠
- 常時表示されている説明文「分類タグはメタデータ保護のため編集対象外です」はノイズなので、構造化タグチップ（`cv/` 等）のツールチップに移す

さらに「編集モード」自体を廃止し、タグチップ列の末尾に常設の「+」を置いてポップオーバーで追加・チップの×で削除、変更は即 `patchWork` 発行にする。保存/キャンセルの往復が消え、T3の整理とも噛み合う（`patchWork` はタグ全置換なので即時発行でもセマンティクスは変わらない。`LibraryView.tsx:148-165`）。

## T3. 作品詳細（WorkDetail）の再構成（優先度: 高)

表示と編集を分離する。

- インライン編集（タイトル/タグの2系統）を廃止。ヒーロー右上に単一の編集エントリ（鉛筆アイコン）を置き、ポップオーバー（またはモーダル）にタイトル・タグ編集を集約する。同時展開・保存2組問題が構造的に消える
- アクション行は「再生」「状態」「その他」に整理する:
  - `[▶ 続きから 1:33]`（primary pill・再開位置を短く内包）
  - `[↺]` 最初から（アイコン）
  - `[♥]` ブックマーク（現状どおり即時トグル）
  - `[⋯]` メニュー（DLsiteを開く / 編集 / 将来の操作）
  - 現状の「いらっしゃいませ・1:33 から再開」の説明文はpillに吸収して削除
- キッカー行（登録済 / 追加日 / 最終再生）は残すが、ラベル文字を減らしアイコン + mono数字で整える

併せて構造リファクタ: 544行の `PreviewPane.tsx` に同居する `WorkDetail` / `AxisLanding` / `SmartFolderView` / `EmptyPreview` を `features/library/ui/preview/` 配下に分割する（挙動変更なしの純リファクタ）。

## T4. 再生UIの磨き込み（優先度: 中）

2層構成は維持したうえで:

- バグ修正: バー表示中に `.mle-bar1__body` が下のコンテンツのクリックを奪う問題。バーの当たり判定を実表示領域に限定し、スクロールコンテナ側にバー高さぶんの padding-bottom を確保する
- バーにクリック可能なシークバーを昇格させる（現状は下端の細い進捗線のみで操作不可）。経過/総時間の表示もバーに出す。prev/next をバーに置くかはお兄ちゃんの判断待ちでOK（置かない場合もポップアップで完結はしている）
- `FullScreenPlayer.tsx` の全面インラインstyle（`:55-189`）をCSSクラス/Tailwindへ移行する。ボタン共通化（T1）を全画面プレイヤーにも波及させる前提整備

## T5. こまごまとした磨き（優先度: 低〜中）

- アイコンボタンの余白基準を T1 のサイズ対で統一（例: トップバー30px箱の14pxアイコンは小さい → 16pxへ）
- スマートフォルダーのルール値の整形表示（`長さ ≥ 3600` → `長さ ≥ 1:00:00`）。表示層だけの修正
- タグ結果グリッドのカードにメタ（サークル・尺）を1行追加
- 未配線ボタン（ビュー切替 `AddressBar.tsx:89-111`、履歴/ピン留め `LeftNav.tsx:50-59`、条件を追加 `PreviewPane.tsx:452`）は、実装まで disabled + 「近日実装」ツールチップにして無反応をなくす
