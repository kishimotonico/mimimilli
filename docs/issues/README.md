# issues 索引

日々の作業記録・提案・レビューの置き場。命名は `yyyy-mm-dd-summary-description.md`（AGENTS.md 参照）。

各ファイルは本文2行目に `Status:` 行を持つ（ファイル名にはステータスを含めない。リネームによるリンク切れを避けるため）。

- `done` — 対応完了（クローズ理由があれば付記）
- `wip` — 対応中
- `todo` — 対応未着手（調査・提案のみ完了）
- `note` — 対応を前提としない記録
