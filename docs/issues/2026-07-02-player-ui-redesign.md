# 再生UI再設計（バー / ポップアップ / 全画面の3層構成）

Status: done — L⇄R・A-Bリピートの UI 配線は BACKLOG.md へ

## 背景

現状の再生UIは `TransportBar`（下部バー）1本に再生・シーク・ループ・音量・±10秒・全画面展開を詰め込んでおり、窮屈で拡張性がない。DLsite Play の再生UI（下部張り付きバー + 右下ポップアップ）を参考に、役割の異なる複数の再生UIへ再構成する。参考スクショ: `tmp/2026-07-02_23h45_43.png`（ポップアップ）、`tmp/2026-07-02_23h46_48.png`（バー）。

また `playbackRate` / `channelSwap` / `abRepeat` はエンジン実装（atoms / usePlayer / audioEngine）のみ存在し、現状どのUIからも操作できない。これらの露出先も本再設計で位置づける。

## 方針

3層構成とし、下ほど多機能にする。

| 層                             | 役割                                       | 機能量               |
| ------------------------------ | ------------------------------------------ | -------------------- |
| 画面下張り付きバー             | 再生中の常駐表示。邪魔にならないこと最優先 | 最小                 |
| 右下ポップアップ               | 日常操作（音量・トラック移動など）の置き場 | 使いやすさ重視で厳選 |
| 全画面再生ページ（集中モード） | じっくり聴く・ニッチ機能の置き場           | 全機能               |

- ポップアップに全機能は載せない。載せない勇気を持つ
- ニッチ機能（channelSwap / abRepeat 等）は全画面側に置く
- バーは今後の作り込みで多機能化していく余地あり（現時点では最小構成）
- 今回の実装スコープはバー + ポップアップ。全画面（集中モード）はそのうちやる

## 仕様: 画面下張り付きバー

- 未再生時は表示しない。プレースホルダー（「作品をクリックして再生」等）も出さない。再生開始時に画面下からスライドインする（motion は mimimilli 正典に従う）
- 構成: カバー画像サムネイル + トラック名 + 作品名 + 再生/一時停止ボタン + 展開アフォーダンス（右端に ^ アイコン）
- ボタンは再生/一時停止のみ。ミスクリックで展開しないよう、ボタンはサイズ大きめ + 周囲に余白を取る
- トラック名・作品名は長くなりがちなので ellipsis で省略し、ホバーで全文を title 表示する
- シークバーは要素全体の下辺を使う（DLsite Play 風。スライダーではない）
  - 表示は 4〜6px 程度（DLsite Play より太め、見やすさ重視）。ホバー中はさらに太く表示する
  - 当たり判定は表示より広く取る。バーは画面最下部に張り付くため、当たり判定をウィンドウ最下端まで含めると、マウスを画面下端に投げるだけで当たる（Fitts の法則）
  - ホバー時、マウス位置の時刻を吹き出し型ツールチップで `12:34` 形式表示。クリックでその位置へジャンプ
  - ドラッグでのスクラブにも対応する
- シークバーと再生ボタンを除く領域のクリックで、右下ポップアップUIへ切り替わる
  - 切替は瞬時に差し替えず、バーが右下へ収束する（またはそれに準ずる）トランジションで繋ぎ、再生ボタンの位置移動を目で追えるようにする（motion は mimimilli 正典に従う）
- ×（閉じる）ボタンは置かない。停止は一時停止で代替する（一時停止中もバーは残る。レジューム再生とも整合）
- バー表示中はコンテンツのスクロール領域に padding-bottom を追加し、リスト最下部の項目が隠れないようにする

## 仕様: 右下ポップアップ

- 位置は右下固定。幅 320〜360px 程度（コンテンツとの被りを抑える）
- 構成（上から）:
  - 折りたたみボタン（下矢印）: バーUIへ戻る
  - 大きめのカバー画像
  - シークバー（こちらはスライダー形式でよい）+ 現在時刻 / 総時間
  - トラック名 + 作品名
  - コントロール: 前トラック / 再生・一時停止 / 次トラック
  - 音量スライダー
  - ループ切替、再生速度（コンパクト表示。例: `1.0x`）
  - 全画面（集中モード）への展開ボタン
- channelSwap / abRepeat は置かない（全画面側）
- 「再生中の作品を表示」導線を置く（ファイラー上の該当作品へジャンプ）。別の場所を閲覧中に今聴いている作品へ戻る手段がないと不便
- 音量アイコンのクリックでミュートのトグル

## 仕様: 全画面再生ページ（集中モード）

そのうちやる。役割だけ先に確定する。

- 集中して聴くためのページ。大きなカバー・トラックリストに加え、全機能を露出する
- playbackRate の詳細調整、channelSwap、abRepeat はここが正式な置き場
- 構築するまでは既存 `FullScreenPlayer` をほぼそのまま残し、ポップアップから遷移できるようにする（機能後退を避けるため）

## 共通仕様

- キーボードショートカット: Space = 再生/一時停止、← / → = ±10秒（入力フィールドにフォーカスがない場合のみ）。バーから ±10 秒ボタンを撤去する代替
- バー / ポップアップのどちらを使っていたかは記憶し、次回再生時に復元する（localStorage）
- 実装は既存の高頻度 atom 設計（`playerCurrentTimeAtom` / `playerDurationAtom` を購読側で絞る）に乗せる。`TransportBar` はバーUIへ作り替え、ポップアップは新規コンポーネント

## 実装メモ（2026-07-03）

実装は Sonnet 5 サブエージェントに委譲し、オーケストレーター側で agent-browser により実機検証した。

- 新規: PlayerDock.tsx（外枠・モード切替・AnimatePresence）、BarContent.tsx、PopupContent.tsx、useSeekDrag.ts
- 変更: atoms.ts（playerUiModeAtom, atomWithStorage `mimimilli:playerUiMode`）、usePlayer.ts（toggleMute）、App.tsx、AppShell.tsx（バーを fixed オーバーレイ化）、useGlobalShortcuts.ts（←/→ ±10秒）、shell.css
- バー⇄ポップアップの切替は motion の layout / layoutId（再生ボタン共有）による FLIP アニメーション
- 並行セッションが実装した playbackError 表示は BarContent に統合済み
- 旧 TransportBar.tsx は未参照だが削除保留（並行セッションが編集中のため）

### 検証結果

- agent-browser で確認: 未再生時プレースホルダーなし / バー出現 / 下辺シークバーのツールチップ・クリックジャンプ / バークリック→ポップアップ / 折りたたみ→バー / 前後トラック・音量・ミュート・ループ・速度巡回（1.0x→1.25x）/ 全画面展開 / 「再生中の作品を表示」（?work=RJ501002 復帰）/ Space・←→ショートカット / padding-bottom 84px
- ポップアップがビューポートより高い場合に上へはみ出すバグを検出→修正（max-height: calc(100dvh - 32px) + カバーのみ縮む flex 構造）。高さ 577px / 400px で全コントロール操作可能を確認
- `pnpm check` 全通過。`pnpm test` は client 65/67（失敗2件は並行セッションの playbackError 作業由来の usePlayer.test.ts で、本実装とは無関係）
- 既知の見た目課題: dev 限定で TanStack Query Devtools のボタンがバー右端の展開アイコンに重なる

## 実装メモ 第2ラウンド（2026-07-03）

ユーザーレビューを受けた再調整。並行セッションの再生エラーハンドリング実装（audioEngine の onError / playbackError / StrictMode 対応）は取り込みで確定。

- 切替アニメーション: layout FLIP と再生ボタンの layoutId 共有を廃止し、AnimatePresence の enter/exit（バー=下からスライド、ポップアップ=右下起点 scale+fade）に変更。カバーの歪み・ボタン飛来を解消
