# ローカル音声作品管理・再生アプリ UI設計書

> **⚠️ 廃止予定（2026年5月）:** 本書は旧グリッド/テーブル UI コンセプトに基づいており、現在は参照非推奨。UIデザインの正典は `design_handoff_mimimilli_library/` (mimimilli) に移行済み。採用方針は `docs/design-brief.md` §10 を参照。

> 本書は要件定義書 v4 に基づくUI設計の詳細ドキュメントである。フロントエンド `src/` は現在再構築中であり、本書をデザインの拠り所として実装を進める。

## 1. 全体レイアウト

### 1.1 構成

アプリは以下の3つのレイヤーで構成される。

**ヘッダー（固定、高さ52px）:** アプリロゴ、検索バー、表示モード切り替え、グリッドサイズ調整、再読み込みボタン、設定ボタン。

**メインエリア（可変）:** 検索条件バー＋ライブラリ一覧（グリッド/テーブル）が表示される。作品フルビューに遷移した場合はメインエリア全体が置き換わる。

**プレイヤーバー（固定、高さ72px）:** 画面下部。再生中のみ表示。

### 1.2 カラースキーム

ダークテーマを基調とする。長時間のASMR視聴に適した、目に優しい配色。

- 背景: `#13132a`（メインエリア）、`#1a1a2e`（ヘッダー）、`#141426`（プレイヤーバー）
- サーフェス: `#1c1c32`（パネル）、`#252540`（入力欄・ボタン背景）
- ボーダー: `#2a2a40`
- テキスト: `#e2e2f0`（プライマリ）、`#888`（セカンダリ）、`#555`（ディスエイブル）
- アクセント: `#5b8def`（選択・アクティブ・リンク）
- エラー: `#e53e3e`

## 2. ヘッダー

### 2.1 構成要素（左から右）

**アプリロゴ:** 🎧 AudioLib（固定幅）

**検索バー:** テキスト入力。プレースホルダー「作品名・タグで検索...」。最大幅420px。左端に🔍アイコン。入力内容は検索条件バーにチップとして反映される。

**表示モード切り替え:** ▦（グリッド）/ ☰（テーブル）の2ボタントグル。アクティブなモードは青色でハイライト。テーブル表示はフェーズ2で実装するが、UIにはトグルを配置しておく。

**グリッドサイズ調整:** 「− [S/M/L/XL] +」のコントロール。4段階でカバー画像サイズを変更（120px / 160px / 200px / 260px）。グリッドモード時のみ表示。

**再読み込みボタン:** ↻アイコン。押すとライブラリの再読み込みを実行する。

**設定ボタン:** ⚙アイコン。設定モーダルを開く。

## 3. 検索条件バー

ヘッダーの直下、ライブラリ一覧の上部に配置。

### 3.1 構成要素

**アクティブな検索条件（左寄せ）:** 現在適用されている検索ワード・タグフィルターをチップ形式で表示する。各チップに×ボタンで個別削除が可能。条件がある場合のみ「すべてクリア」リンクを表示。

**結果件数:** 「N 作品」の形式で表示。

**並び替えドロップダウン（右寄せ）:** 「並び替え: [現在の条件]」ボタン。クリックでドロップダウンメニューを表示。フェーズ1では追加日（新しい/古い順）、タイトル順（昇順/降順）、再生時間順（長い/短い順）に対応。ドロップダウン末尾に「💾 プリセットとして保存...」を将来機能として表示（グレーアウト）。

## 4. ライブラリ一覧画面

### 4.1 グリッド表示（フェーズ1）

カバー画像をグリッドで表示する。カードの構成は以下の通り。

**カバー画像:** 正方形。サイズはグリッドサイズ設定に従う。

**オーバーレイアイコン:** エラーのある作品はカバー右上に赤い「!」バッジ。ブックマーク済み作品は左上に🔖アイコン（将来実装）。

**テキスト情報:** カバー画像の下にタイトル（1行、省略記号）、トラック数と合計再生時間（「6曲 · 1:10:13」形式）。最近再生した作品には「● 最近再生」インジケーター（将来実装）。

### 4.2 テーブル表示（フェーズ2）

列構成: ミニカバー画像（40px）、タイトル、タグ（最大3つ+残数）、曲数、合計再生時間。再生中の作品行は青色ハイライト。

### 4.3 クリック操作

**シングルクリック:** クイックビュー（右パネル）を表示。既に同じ作品のパネルが開いている場合は閉じる。

**ダブルクリック:** 作品フルビューに遷移。シングルクリックとの干渉を防ぐため、シングルクリックに200msの遅延を入れ、ダブルクリック時にはシングルクリックの発火をキャンセルする。

## 5. クイックビュー（右パネル）

### 5.1 概要

ライブラリ一覧の右側にオーバーレイとしてスライドイン表示される。幅370px。グリッドの幅は変化しない（パネルがグリッドの上に重なる）。パネル外クリックで閉じる。

### 5.2 構成

**パネルヘッダー:** 「クイックビュー」ラベル、「詳細を開く ↗」ボタン（フルビューへ遷移）、✕閉じるボタン。

**作品情報エリア:** カバー画像（90px）、タイトル、トラック数＋合計再生時間、最終再生日時（将来）、エラー表示、URLリンクボタン。

**タグエリア:** タグ一覧。Annotatedタグ（`cv/xx` 等）は青色で視覚的に区別する。各タグに削除ボタン、末尾に「+ 追加」ボタン。

**トラックリスト:** defaultPlaylistのトラック一覧。各トラックに番号、タイトル、再生時間。クリックで再生開始。再生中のトラックは青色ハイライト。

### 5.3 URLリンクボタン

作品に紐づく複数のURLを個別のボタンとして表示する。各ボタンにはラベル（DLsite, FANZA, Twitter等）と↗アイコンを表示。クリックで外部ブラウザで開く。URLがない作品ではボタンを非表示。

## 6. 作品フルビュー

### 6.1 概要

画面全体（ヘッダー下）を使って作品の詳細を表示する。ライブラリ一覧を置き換える形で表示される。

### 6.2 レイアウト

**トップバー:** 「← 戻る」ボタン（ライブラリ一覧に戻る）、作品タイトル、URLリンクボタン。

**左サイドバー（幅300px）:** カバー画像（260px）、タイトル、トラック数＋合計再生時間、ブックマーク状態（将来）、最終再生日時（将来）、エラー表示、タグ一覧（編集可能）、メタ情報（パス、追加日、URL）。

**右メインエリア:** タブ切り替え方式。

### 6.3 トラックタブ

defaultPlaylistのトラック一覧を広いエリアで表示する。先頭に「▶ すべて再生」ボタンと合計再生時間。各トラックは番号、タイトル、再生時間。クリックで再生開始。再生中トラックは青色ハイライト。

### 6.4 ファイルタブ（フェーズ2）

作品フォルダーの物理構造をフォルダーツリー形式で表示する。フォルダーは展開/折りたたみ可能。フェーズ1ではタブのみ表示しグレーアウト。

**ファイル種別ごとの操作:**

- **音声ファイル:** クリックで再生。「▶ 再生」ラベル表示。トラックリストに含まれない物理ファイルもここから再生可能。
- **画像ファイル:** クリックでタブ上部にプレビューエリアが展開。ファイル名、解像度、サイズを表示。
- **テキストファイル / PDF:** 扱いは別途検討。

各ファイルにはアイコン（📁/🔊/🖼/📄）、ファイル名、サイズを表示する。

## 7. プレイヤーバー

### 7.1 レイアウト（左から右）

**左エリア（カバー＋情報）:** カバー画像（48px）、トラックタイトル、作品タイトル、▲展開アイコン。クリックでフル画面プレイヤーに展開。

**中央エリア（コントロール＋シーク）:** 上段にコントロールボタン（🔁ループ、⏮前へ、-10s、▶再生/一時停止、+10s、⏭次へ）。下段にシークバー（maxWidth: 680px）。左右に現在時刻と合計時間を表示。ASMR作品では再生時間が重要なため、シークバーは十分な幅を確保する。

**右エリア（音量）:** 🔊アイコン＋音量バー。

### 7.2 フローティングモード（フェーズ2）

画面右下にフローティング表示。ドラッグで位置変更可能。固定バーとの切り替えはプレイヤーバー上のトグルボタンで行う。フローティング時は画面下部の領域を解放する。

## 8. フル画面プレイヤー

### 8.1 構成

プレイヤーバーのクリックまたは▲アイコンで画面全体に展開する。下にスワイプまたは「▼ 閉じる」ボタンでバーに戻る。

**上部:** 「▼ 閉じる」ボタン、「再生中」ラベル、URLリンクボタン。

**中央:** カバー画像（220px）、トラックタイトル（大）、作品タイトル、シークバー（maxWidth: 560px、つまみ付き）、コントロールボタン（ループ、前へ、-10s、▶再生/一時停止、+10s、次へ、すべてプレイヤーバーより大きいサイズ）、音量。

**下部:** トラックリスト（スクロール可能、maxHeight: 180px）。再生中トラックをハイライト。

## 9. 設定モーダル

### 9.1 構成

画面中央にモーダル表示。背景をオーバーレイで暗くする。

**ルートフォルダー:** 現在のパス表示＋「変更」ボタン。

**ライブラリスキャン:** 「フルスキャン実行」ボタン、スキャン中はスピナー表示、前回スキャン日時の表示、説明テキスト。

**バージョン情報:** アプリ名とバージョン。

## 10. 新規作品検出ポップアップ

### 10.1 概要

スキャンで新しい作品候補が検出された際に表示されるダイアログ。

### 10.2 構成

**作品情報:** カバー画像のプレビュー（あれば）、フォルダー名/ファイル名、パス、検出されたファイルの概要（「MP3ファイル × 8 · カバー画像あり」等）。

**タイトル入力:** デフォルトはフォルダー名（またはRJコード）。ユーザーが編集可能。

**タグ情報:** フェーズ1では「フェーズ2で自動取得予定」のプレースホルダーを表示。フェーズ2ではスクレイピング結果（サークル名、CV、ジャンルタグ等）をここに表示し、ユーザーが確認・編集してから登録する。

**アクション:** 「登録する」（メタファイルを生成しDBに追加）、「スキップ」（今回は登録しない）。複数の新規作品がある場合は1件ずつ順番に表示する。

## 11. 画面遷移

```
初回起動 → [セットアップ] → スキャン完了 → [ライブラリ一覧]
通常起動 → [ライブラリ一覧]

[ライブラリ一覧] ─ シングルクリック ──→ [クイックビュー] 表示（オーバーレイ）
[ライブラリ一覧] ─ ダブルクリック ──→ [作品フルビュー] に遷移
[クイックビュー] ─ 「詳細を開く」 ──→ [作品フルビュー] に遷移
[クイックビュー] ─ パネル外クリック ─→ [クイックビュー] 閉じる
[クイックビュー] ─ トラッククリック ─→ 再生開始 → [プレイヤーバー] 出現
[作品フルビュー] ─ 「← 戻る」 ────→ [ライブラリ一覧] に戻る
[作品フルビュー] ─ トラッククリック ─→ 再生開始 → [プレイヤーバー] 出現

[プレイヤーバー] ─ クリック ────────→ [フル画面プレイヤー] に展開
[フル画面]       ─ 「▼ 閉じる」 ───→ 元の画面 + [プレイヤーバー]

[ライブラリ一覧] ─ ⚙ ──────────→ [設定モーダル]
[設定モーダル]   ─ スキャン完了 ──→ [新規作品検出ポップアップ]（該当がある場合）

[プレイヤーバー] は再生中、ライブラリ一覧/フルビューのどちらでも下部に常駐
[フル画面プレイヤー] 表示中はプレイヤーバーは非表示
```

## 12. 将来の拡張を見据えた設計上の考慮

**ブックマーク:** カバー画像上のアイコン表示、フルビューのサイドバーにブックマーク状態表示の場所を確保。DBにブックマークフィールドを持つ。

**最近再生した作品:** グリッドカードに「最近再生」インジケーター表示。並び替えの選択肢に「最近再生した順」を追加。DBに最終再生日時フィールドを持つ。

**検索プリセット:** 並び替えドロップダウンに「プリセットとして保存」の導線を用意。プリセットはDB内に保存し、検索条件バーからワンクリックで適用可能にする。

**複数URL:** メタファイルの `urls` フィールドを配列として設計済み。UI上は各URLを個別のボタンとして表示し、1〜5件程度を想定。

**フォルダービュー:** 作品フルビューのファイルタブとして画面全体の幅を使って表示する。クイックビューのタブではなく、フルビューの中に配置する設計。

**フローティングプレイヤー:** 固定バーと同じコンポーネントの表示モード違いとして実装する。切り替えトグルをプレイヤーバー上に配置する予定。
